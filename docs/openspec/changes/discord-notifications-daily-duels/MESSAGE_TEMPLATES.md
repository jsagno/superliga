## Variables de Plantilla

- `{{player}}` → Nombre del ganador (para VICTORIA) o del perdedor (para DERROTA)
- `{{opponent}}` → Nombre del perdedor (para VICTORIA) o del ganador (para DERROTA)
- `{{score}}` → Puntuación final (ej., "2-0" o "2-1")
- `{{zone}}` → Nombre de la zona (opcional, para contexto)

## Lógica de Selección

```python
import random

WIN_MESSAGES = [...]  # 50+ mensajes
LOSS_MESSAGES = [...]  # 50+ mensajes

def get_message_template(result_type):
    """Devuelve plantilla de mensaje aleatorio para VICTORIA o DERROTA"""
    if result_type == "WIN":
        return random.choice(WIN_MESSAGES)
    else:
        return random.choice(LOSS_MESSAGES)
```

## Formato de Mensaje

```python
def format_message(template, player, opponent, score):
    """Formatea la plantilla con datos reales del jugador"""
    return template.replace("{{player}}", player) \
                   .replace("{{opponent}}", opponent) \
                   .replace("{{score}}", score)
```

## Notas para Desarrolladores

- Los mensajes deben ser DIVERTIDOS, nunca ofensivos
- Evitar referencias a eventos específicos del mundo real
- Mantener los mensajes bajo 100 palabras para legibilidad
- Usar emojis de Discord generosamente para diversión
- Probar el formato de mensaje en Discord antes de desplegar
- Los mensajes se aleatorizan cada vez que se vincula una batalla
- El mismo jugador jugando dos veces en un día obtendrá mensajes diferentes cada vez
